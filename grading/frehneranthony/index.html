Django: A Python MTV Framework

A. Frehner

Where did Django come from?

I tried to write something, but decided that Wikipedia had stated it more clearly and concisely than I could. Here's what Wikipedia says:

Django was born in the fall of 2003, when the web programmers at the Lawrence Journal-World newspaper, Adrian Holovaty and Simon Willison, began using Python to build applications. It was released publicly under a BSD license in July 2005. The framework was named after guitarist Django Reinhardt.

In June 2008, it was announced that a newly formed Django Software Foundation (DSF) would maintain Django in the future.

Wikipedia Article on Django

Also, MTV stands for "model, template, view", which is the Django interpretaion of MVC (model, view, controller). In Django, the model is used for database interactions, the view is used for business logic, and the template is a dynamic html document intermixed with the template language.

What are some good things about Django?

Database interactions in Python

Once you lay out your database in Django, it allows you to do CRUD functions without switching languages. For example, if you have a person object named John, you can simply write:

John.save()

and that object is now saved in the database. Now, let's say we need to change John's address (he moved). Easy enough:

John.address = "111 W"
John.save()

Look at that. You don't have to worry about UPDATE or INSERT or anything; you just "save" the object and Django handles that for you. But don't worry; if you need to write pure SQL, you still can, too.

Automatic, production-ready admin interface

The admin interface is generally just as large and time consuming to build as the public-facing side of your webpage. Well, fret no more: Django automatically builds your admin site, so you don't need to worry about that.

Clean URLs

Ever visit a website that has URLs with random characters, numbers, and symbols in it? Django makes it so those days are over: your pages will have easy-to-read URLs without any trailing markers like .php or .asp .

Using templates: a great DRY principle

DRY stands for Don't Repeat Yourself. Almost nothing is worse than doing the same thing over and over again. Django's templates allow you to build a base look and layout for your webpage, and then dynamically fill it with content on a per-page basis. This way, you don't have to re-do the html over and over again for each page that you build. A fair warning, however; the language used for the template system is not Python, though it is very simple to learn.
